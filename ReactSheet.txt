━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

// 每个 SSE 块格式：
//   event: <event_type>
//   data: <JSON string>
//
// 事件类型与 data 格式：

event: reasoning
data: "正在分析输入意图并扫描本地权限库..."    // string，RAG 判定阶段的进度提示（可能多条）

event: reasoning
data: "记忆锚点命中 2 条"

event: thinking
data: "用户问的是..."                          // string，模型 CoT 思考流（逐 chunk，拼接展示）

event: content
data: "量子纠缠是两个粒子..."                  // string，正文回复流（逐 chunk，拼接展示）

event: anchor_created
data: {"pattern": "量子纠缠"}                  // G045 function call 创建了新记忆锚点时触发

event: telemetry
data: {
  "platform": "gemini",
  "model_name": "gemini-1.5-pro",
  "input_chars": 15820,
  "output_chars": 950,
  "tool_calls_in_turn": 2
}

event: error
data: "LLM 传输层中断: ..."                    // string，发生错误时

event: done
data: "[DONE]"                                 // 流结束标志

// 前端处理示例：
//   const es = new EventSource(...)  ← 或用 fetch + ReadableStream
//   es.addEventListener('content', e => fullText += e.data)
//   es.addEventListener('thinking', e => thinkingText += e.data)
//   es.addEventListener('reasoning', e => reasoningSteps.push(e.data))
//   es.addEventListener('telemetry', e => displayUsage(JSON.parse(e.data)))
//   es.addEventListener('done', () => es.close())

// GET 返回：
{
  "id": 88,
  "uid": "kf_88",
  "title": "量子力学笔记",
  "topic": "Physics",
  "tags": ["量子", "波函数"],
  "keywords": ["薛定谔方程", "叠加态"],   // 用户维护的 L2 关键词（可 PATCH 修改）
  "abstract": "本文介绍量子力学基本原理...",
  "status": "active",
  "source_type": "obsidian_md",
  "project": 3
}

// PATCH 请求体（所有字段可选）：
{
  "abstract": "修改后的摘要内容",    // 修改此字段 → 后台异步触发重向量化（pgvector）
  "keywords": ["新关键词1", "新关键词2"]  // 修改此字段 → 同步入库，无需重向量化
}

// PATCH 响应 (200)：
{ "msg": "已保存，摘要重向量化已在后台启动。", "updated": ["abstract", "keywords"] }
// 或（仅改了 keywords）：
{ "msg": "已保存。", "updated": ["keywords"] }

// 检索行为说明（前端可在 tooltip 或帮助文本中展示）：
//   title（文章标题）= L1 关键词：命中时必定召回，默认返回摘要，
//                      若向量相似度同时 ≥ 0.75 则升级为全文
//   keywords 字段    = L2 关键词：用户自定义，命中时必定召回并返回全文
//   向量检索         = 基于 abstract 的语义召回，相似度 < 0.52 时过滤
